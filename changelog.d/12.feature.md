## Multi-Repo Skill Composition via `--repo` Flag

`dot-ai skills generate` now accepts a `--repo <url>` flag, enabling skills to be composed from multiple independent git repositories without clobbering each other. Previously, running the command twice was useless because each invocation wiped all existing skill files — making it impossible to maintain separate sources with independent credentials (for example, org-wide shared skills on GitHub alongside private team skills on an internal GitLab).

Each invocation now owns exactly its slice of the output directory. Generated skill files carry a `source:` field in their YAML frontmatter recording which repository they came from, and a subsequent generate for that source only deletes and rewrites files from that same source — files from other sources are left untouched. Cross-source name collisions are resolved by first-source-wins: if a skill name is already provided by another repository, the incoming skill is skipped and a warning is logged so the conflict is visible. Concurrent invocations (for example, multiple agent hooks firing in parallel) are serialized via an exclusive file lock on the output directory; if the lock cannot be acquired within 5 seconds the command exits with a clear error rather than silently corrupting the directory.

Compose skills from multiple sources by configuring one agent hook per source, each passing its own `--repo` URL and credentials via environment variables:

```bash
# ~/.claude/hooks/session-start.sh (example three-source composition)
dot-ai skills generate --repo https://github.com/orgA/shared-skills
DOT_AI_GIT_TOKEN=$TOKEN_B dot-ai skills generate --repo https://gitlab.corp/team/skills
dot-ai skills generate   # uses DOT_AI_USER_PROMPTS_REPO env var (default source)
```

Existing installations (skill files without `source:` frontmatter) are handled automatically: the default env-var path reclaims untagged legacy files on the next generate run, and a `--repo` invocation leaves them intact until the env-var-repo generate cleans them up.

See the [Skills Generation Guide](https://devopstoolkit.ai/docs/mcp/guides/skills-generation) for full configuration details and examples.
