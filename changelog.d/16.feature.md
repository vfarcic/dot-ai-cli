## Per-Request Path, Branch, and Credential on the Prompts Repo Override

`dot-ai skills generate` now supports two new flags — `--repo-path <subdir>` and `--repo-branch <branch>` — alongside automatic forwarding of the CLI host's `DOT_AI_GIT_TOKEN` as the `X-Dot-AI-Git-Token` request header when `--repo` is in use.

Previously, `--repo <url>` (added in PRD #15) could only point at a repo root on its default branch, authenticated solely by the *server's* single git token — making a secondary source in a subdirectory, on a non-default branch, or in a different auth realm unreachable from the CLI.

The three new capabilities are optional and additive:

- **`--repo-path <subdir>`** — tells the server to resolve skills from a subdirectory within the repo (e.g. `--repo-path skills/`), matching the layout an env-var repo uses via `DOT_AI_USER_PROMPTS_PATH`.
- **`--repo-branch <branch>`** — pins the clone to a specific branch instead of the server's default.
- **`DOT_AI_GIT_TOKEN` forwarding** — when set in the CLI environment and `--repo` is active, the token is sent as the `X-Dot-AI-Git-Token` header on every override request (list, render, and refresh). It is never logged, persisted, or echoed by the server. On cross-host redirects the header is automatically stripped before following the redirect.

Backward compatibility: with none of the new inputs supplied, the CLI behaves byte-identically to the `#15` `--repo` behavior; without `--repo` it is fully unchanged.

Configure per-source auth by setting `DOT_AI_GIT_TOKEN` in the hook's environment:

```bash
# ~/.claude/hooks/session-start.sh
DOT_AI_GIT_TOKEN=$TOKEN_INTERNAL dot-ai skills generate --repo https://git.corp/team/skills --repo-path skills/ --repo-branch main
dot-ai skills generate   # default env-var source, unaffected
```

See the [Skills Generation Guide](https://devopstoolkit.ai/docs/mcp/guides/skills-generation) and [Configuration Reference](https://devopstoolkit.ai/docs/mcp/setup/configuration) for full details.
