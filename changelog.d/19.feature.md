## User-Level Skill Installs with `--global`

`dot-ai skills generate` now accepts a `--global` flag (with `--agent claude-code`) that targets the user-level `~/.claude` layout instead of the project-local one, so a shared skill catalog is available in **every** project rather than one at a time. Previously `--install-hook` only supported per-project setups: the hook was hard-coded to the project-local `.claude/settings.json`, and pairing `--install-hook` with a custom `--path` was rejected outright — so onboarding a whole team to an always-fresh global catalog meant hand-editing `~/.claude/settings.json` plus a manual one-shot generate.

With `--global`:

- The `SessionStart` hook is written to `~/.claude/settings.json` (resolved via the home directory) instead of `./.claude/settings.json`.
- Without `--path`, skills default to `~/.claude/skills` (the shared "global catalog") instead of `./.claude/skills`.
- The `--install-hook` / `--path` conflict is lifted (project mode keeps it), so a global hook can pair with a custom output directory such as `~/.claude/commands`.

Team onboarding becomes a single idempotent command:

```bash
dot-ai skills generate --agent claude-code --install-hook --global
```

`--global` composes with every source flag (`--repo`, `--repo-fetch`, `--repo-dir`): the named source is round-tripped into the stored command and the hook lands in `~/.claude/settings.json`. Only one dot-ai `SessionStart` hook is kept at a time — running `--install-hook` again replaces the existing dot-ai hook rather than adding a second, so a later install wins (the generated skill files stay source-scoped and still accumulate across per-source runs). Installing a global hook merges with and preserves any unrelated content already in `~/.claude/settings.json`, and re-running is idempotent. The stored hook command round-trips `--global` itself (not the resolved path), so it re-resolves `~/.claude/skills` against `$HOME` on every firing and stays portable across machines with different home directories; a custom `--path` is stored as the absolute path the shell already expanded. `--global` without `--install-hook` is also valid — it is the "write to the global catalog" mode. A missing or unreadable home directory surfaces a clear error rather than writing to a surprising location.

As a bonus, `~/.claude/skills` is natively discovered by **opencode** (from `~/.claude/skills/*/SKILL.md`), so a single `--global` install lights up both Claude Code and opencode for skill discovery with no opencode-specific work.

Backward compatibility is preserved: a run **without** `--global` is byte-identical to before — same conflict guard, same `.claude/settings.json` and `.claude/skills` locations, and the same stored hook command string. `--global` is purely additive and opt-in.
